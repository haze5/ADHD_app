FocusGuard - ADHD专注守护 需求文档

📋 文档信息

项目名称: FocusGuard - ADHD专注守护

版本: v1.0

创建日期: 2025年11月

目标用户: ADHD人群，特别是做事拖延、启动困难者

🎯 产品概述

产品愿景

为ADHD用户提供一个理解他们神经特质、结合强制约束与情感支持的专注力训练伙伴，帮助克服启动困难和任务拖延问题。

核心价值主张

理解而非批判: 基于ADHD神经科学设计

强制但友好: 通过游戏化机制实现必要约束

陪伴式成长: 动漫伙伴提供持续的情感支持

🏗️ 功能模块详述

模块一：智能任务管理系统

1.1 日常计划管理

Plain Text
功能描述:
- 用户设置每日必须完成的核心任务
- 任务属性：名称、描述、预估难度、时间要求
- 支持任务分类：工作、学习、生活、健康等

ADHD友好设计:
- 极简任务添加界面（不超过3步）
- 语音输入快速创建任务
- 预设任务模板库

1.2 强制锁机机制

Plain Text
惩罚规则:
- 未完成任务 → 次日锁机30分钟
- 锁机时间段：7:00-9:00 和 20:00-24:00
- 连续未完成 → 锁机时间累积（30min × 连续天数）

技术实现:
- 安卓：Device Policy Manager
- iOS：Screen Time API + 强制提醒模式
- 锁机期间：仅允许紧急通话

1.3 任务难度智能识别

Plain Text
难度分级逻辑:
高难度: 包含"简历"、"面试"、"报告"、"项目"等关键词
中难度: 包含"学习"、"阅读"、"整理"等关键词  
低难度: 包含"吃饭"、"运动"、"洗漱"等关键词

用户可手动调整难度评级

模块二：灵感捕捉系统

2.1 快速记录

Plain Text
记录方式:
- 悬浮按钮随时触发
- 语音转文字输入
- 支持图片、链接附加

关联逻辑:
- 可选关联到具体任务
- 未关联的灵感进入"灵感池"
- 标签系统自动分类

2.2 灵感管理

Plain Text
查看方式:
- 按任务关联查看
- 时间线视图
- 思维导图模式（v2.0）

模块三：动漫监督伙伴系统

3.1 生命值机制

Plain Text
生命值规则:
- 初始值: 10/10
- 退出任务: -1
- 未完成任务: -2
- 完成任务: +2
- 连续完成: 额外奖励

视觉表现:
- 左上角常驻闪电图标
- 颜色变化：绿色(8-10) → 黄色(5-7) → 红色(1-4)

3.2 行为监督

Plain Text
监督场景:
1. 任务开始 → 伙伴出现鼓励
2. 应用切换检测 → 及时提醒
3. 长时间不操作 → 检查状态
4. 生命值低 → 增加关注频率

3.3 反思引导系统

Plain Text
触发条件: 生命值归零

引导流程:
1. 数据展示
   - 最近24小时手机使用时长
   - 各应用使用分布
   - 专注时段分析
   - 任务完成率统计

2. 情感引导对话
   - "你现在是什么感受？"（情绪识别）
   - "你觉得是什么阻碍了你？"（原因分析）
   - "如果重来一次，你会怎么做？"（解决方案）
   - "明天我们可以尝试什么新策略？"（行动计划）

模块四：智能任务推荐

4.1 推荐算法

Plain Text
推荐策略:
- 难度交替：高 → 低 → 中 → 高
- 时间匹配：当前时段适合的任务类型
- 能量水平适配：根据用户历史表现调整

推荐时机:
- 完成一个任务后
- 用户主动请求建议时
- 检测到用户徘徊时

4.2 任务拆解集成

Plain Text
Goblin Tools API集成:
- 输入任务描述 → 返回步骤分解
- 支持复杂任务自动拆解
- 生成可视化任务流程图

拆解模板:
- 学习类任务：预习 → 重点学习 → 练习 → 复习
- 创作类任务：构思 → 大纲 → 初稿 → 修改
- 整理类任务：分类 → 筛选 → 安置 → 维护

🎮 用户体验流程

首次使用流程

Plain Text
1. 引导介绍动漫伙伴
2. 设置3个核心每日任务
3. 了解生命值系统
4. 权限授权（锁机、使用统计）
5. 完成第一个测试任务

每日使用流程

Plain Text
早上:
- 7:00 今日任务提醒
- 锁机惩罚执行（如有）

白天:
- 任务执行 + 伙伴监督
- 随时记录灵感
- 任务完成获得奖励

晚上:
- 20:00 锁机惩罚执行（如有）
- 每日总结报告
- 明日任务预览

⚠️ 风险与约束

技术风险

iOS锁机限制: Screen Time API功能有限

电池优化影响: 后台服务可能被系统终止

权限获取: 需要用户授予敏感权限

用户体验风险

过度强制: 可能引起用户反感

情感依赖: 用户过度依赖外部激励

挫折累积: 连续失败可能导致放弃

缓解策略

提供灵活的锁机设置选项

设计渐进式难度调整

建立社区支持系统
